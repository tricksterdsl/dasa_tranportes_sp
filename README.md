📌 Problema:
Processos manuais e relatórios difíceis de consultar dificultavam a identificação de atrasos e o acompanhamento operacional em tempo real.

🛠️ Solução:
Construção de um pipeline ETL automatizado que realiza:
- Coleta de dados via requisições HTTP (sem API oficial)
- Tratamento e padronização com pandas
- Persistência em PostgreSQL com histórico incremental
- Disponibilização para dashboards de gestão à vista

⚡ Diferenciais técnicos:
- Execução contínua (~30 segundos)
- Pipeline resiliente a mudanças de estrutura
- Separação em camadas (extract, transform, load)
- Uso de variáveis de ambiente para segurança

📊 Impacto:
- Redução significativa no tempo de análise de rotas críticas
- Maior visibilidade operacional em tempo quase real
